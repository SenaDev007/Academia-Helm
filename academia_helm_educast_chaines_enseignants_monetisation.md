# Academia Helm — Mise à jour du module EduCast : Chaînes enseignants et contenus payants/gratuits

## 1. Mise à jour conceptuelle

Dans EduCast, chaque enseignant doit pouvoir créer sa propre chaîne pédagogique, avec une logique proche de YouTube, mais adaptée au contexte scolaire, aux permissions, à la sécurité des élèves et aux règles de l’établissement.

L’enseignant devient donc un créateur de contenus pédagogiques au sein de l’écosystème Academia Helm.

Il peut :

- créer sa chaîne ;
- personnaliser sa chaîne ;
- publier des vidéos ;
- organiser ses contenus en playlists ;
- créer des séries pédagogiques ;
- décider si une vidéo est gratuite ou payante selon les règles de l’école ;
- suivre les vues, abonnés, revenus éventuels et statistiques ;
- interagir avec ses apprenants selon les paramètres autorisés ;
- proposer des contenus publics internes, privés, gratuits ou premium.

---

## 2. Principe général

EduCast doit fonctionner comme une plateforme de chaînes éducatives internes.

Chaque enseignant peut avoir :

- une chaîne principale ;
- plusieurs playlists ;
- des vidéos pédagogiques ;
- des capsules de révision ;
- des lives / webinaires ;
- des contenus gratuits ;
- des contenus payants ;
- des contenus réservés à certaines classes ;
- des contenus réservés aux abonnés ;
- des contenus validés par l’administration si l’école l’exige.

L’école garde le contrôle global. L’enseignant produit et publie, mais la plateforme applique les règles institutionnelles.

## 5. Parcours de création d’une chaîne

### Étape 1 — Demande de création

L’enseignant clique sur : Créer ma chaîne EduCast.

Il renseigne :

- nom de chaîne ;
- description ;
- matières ;
- niveaux ;
- classes ;
- langue ;
- type de chaîne ;
- image de profil ;
- bannière.

### Étape 2 — Validation selon politique école

Deux modes possibles :

#### Mode libre contrôlé

L’enseignant crée directement sa chaîne. Les contenus sensibles restent soumis à validation.

#### Mode validation obligatoire

La chaîne est créée en brouillon et doit être validée par :

- directeur ;
- responsable pédagogique ;
- responsable communication ;
- administrateur école.

### Étape 3 — Activation

Une fois validée, la chaîne devient active et visible selon les droits définis.

## 9. Vidéos gratuites et vidéos payantes

### 9.1 Principe

L’enseignant peut décider qu’une vidéo soit :

- gratuite ;
- payante ;
- réservée aux abonnés ;
- offerte à une classe ;
- incluse dans un pack ;
- incluse dans un abonnement ;
- accessible après paiement unique ;
- accessible selon autorisation de l’école.

### 9.2 Vidéo gratuite

Une vidéo gratuite peut être accessible :

- à toute l’école ;
- à une classe ;
- à un niveau ;
- aux élèves de l’enseignant ;
- aux parents ;
- à un groupe spécifique.

### 9.3 Vidéo payante

Une vidéo payante nécessite un paiement avant accès.

Elle peut être :

- achetée individuellement ;
- incluse dans un pack de révision ;
- incluse dans une playlist premium ;
- incluse dans un abonnement mensuel ;
- offerte par l’école ;
- prise en charge par le parent.

## 10. Modèles économiques possibles

### 10.1 Paiement par vidéo

Le parent ou l’élève achète une vidéo spécifique.

### 10.2 Pack de vidéos

L’enseignant crée un pack :

- pack révision composition ;
- pack préparation examen ;
- pack chapitre complet ;
- pack méthodologie ;
- pack bilingue ;
- pack vacances.

### 10.3 Abonnement à une chaîne

L’utilisateur s’abonne à la chaîne de l’enseignant pour accéder à certains contenus premium.

### 10.4 Abonnement par matière

L’élève ou le parent paie pour accéder à des contenus premium d’une matière.

### 10.5 Accès sponsorisé par l’école

L’école paie ou débloque certains contenus pour une classe ou un niveau.

### 10.6 Contenus mixtes

Une chaîne peut contenir :

- vidéos gratuites ;
- vidéos premium ;
- lives payants ;
- playlists gratuites ;
- playlists payantes.

## 11. Contrôle de l’école sur les contenus payants

Il est important que l’école garde un contrôle clair.

### Règles recommandées

- L’école décide si la monétisation est activée ou non.
- L’école peut valider les vidéos payantes avant publication.
- L’école peut fixer une commission.
- L’école peut plafonner les prix.

- L’école peut rendre certains contenus obligatoirement gratuits.
- L’école peut contrôler les remboursements.
- L’école peut désactiver la monétisation d’un enseignant.
- L’école peut auditer les revenus.

## 13. Gestion des prix

### Champs recommandés

- prix de base ;
- devise ;
- remise éventuelle ;
- période promotionnelle ;
- prix élève interne ;
- prix parent ;
- prix pack ;
- commission école ;
- part enseignant ;
- taxes éventuelles ;
- statut paiement.

### Exemple de répartition

Pour une vidéo vendue à 1 000 FCFA :

- école : 20 %
- enseignant : 80 %

Ou selon politique :

- école : 30 %
- enseignant : 70 %

L’école doit pouvoir configurer ces règles.

---

## 14. Paiement et accès

### Processus recommandé

1. Le parent ou l’élève clique sur une vidéo payante.
2. Le système affiche le prix et les conditions.
3. L’utilisateur paie via le module Finance/Paiement.
4. Le paiement est validé.
5. L’accès à la vidéo est débloqué.
6. Une facture ou un reçu est généré.
7. L’enseignant voit la vente dans ses statistiques.
8. La direction voit les revenus dans ORION et Finance.

## 30. Règles métier complémentaires

- Un enseignant ne peut créer une chaîne que s’il a la permission.
- Une école peut désactiver la création libre de chaînes.
- Une vidéo payante peut nécessiter une validation administrative.
- L’école peut fixer un prix minimum et maximum.
- L’école peut définir une commission.
- Un contenu acheté doit rester accessible selon la durée définie.
- Un parent ne peut acheter que pour son enfant ou selon les règles de l’école.
- Les remboursements doivent être journalisés.
- Les revenus enseignants doivent être traçables.
- Les vidéos payantes doivent afficher clairement leur prix.
- Les élèves ne doivent pas être exposés à des achats non contrôlés.
- Les commentaires doivent être modérés selon l’âge.
- L’école peut suspendre une chaîne ou un contenu.
- Les statistiques financières doivent être visibles par la direction.
- Les contenus premium ne doivent pas contourner les obligations pédagogiques normales de l’école.

## 31. Conclusion

Avec cette mise à jour, EduCast devient une vraie plateforme de chaînes éducatives, inspirée de YouTube, mais adaptée à l’école.

Chaque enseignant peut créer sa chaîne, publier des contenus, organiser des playlists, proposer des vidéos gratuites ou payantes, créer des packs, suivre ses statistiques et éventuellement générer des revenus.

Mais la différence majeure avec YouTube est le contrôle institutionnel :

- validation ;
- permissions ;
- modération ;
- protection des élèves ;
- intégration Finance ;
- supervision ORION ;
- traçabilité complète ;
- conformité aux règles de l’école.

Ce positionnement donne à Academia Helm une avance forte : l’école ne consomme plus seulement du contenu numérique, elle devient productrice, distributrice et gestionnaire de savoir.
