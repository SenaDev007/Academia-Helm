# Academia Helm — Générateur IA d’Épreuves, Devoirs et Exercices dans l’Espace Pédagogique Enseignant

## 1. Positionnement

Cette fonctionnalité appartient au module Pédagogie, dans l’Espace Pédagogique Enseignant.

Elle permet à l’enseignant de composer automatiquement des épreuves, devoirs, exercices, interrogations, travaux dirigés, fiches d’activités et évaluations à partir :

- de captures d’écran des notions étudiées ;
- de titres de leçons enseignées ;
- de résumés de cours ;
- de compétences ciblées ;
- du niveau scolaire ;
- de la classe ;
- de la matière ;
- du type d’évaluation souhaité ;
- des paramètres pédagogiques définis par l’enseignant.

L’enseignant garde toujours le contrôle final : il génère, modifie, valide, sauvegarde, partage et télécharge.

---

# 2. Objectif

L’objectif est de fournir un assistant de composition pédagogique intelligent, capable d’aider l’enseignant à produire rapidement des supports d’évaluation propres, cohérents et adaptés au programme réellement enseigné.

Le système ne doit pas remplacer l’enseignant.  
Il doit l’assister, accélérer son travail et améliorer la qualité de ses productions.

# 7. Système de Skills personnalisés

## 7.1 Définition

Les Skills sont des profils de génération que l’enseignant peut configurer pour obtenir un rendu constant et professionnel.

Comme dans Claude AI, un Skill peut contenir :

- consignes permanentes ;
- style de rédaction ;
- format attendu ;
- règles pédagogiques ;
- niveau de difficulté ;
- structure de sortie ;
- ton des consignes ;
- contraintes de correction ;
- langue ;
- modèle d’épreuve ;
- exigences propres à l’école.

## 7.2 Exemples de Skills

### Skill : Épreuve Primaire Standard

- consignes simples ;
- exercices progressifs ;
- pas de coefficient ;
- note sur 20 ;
- langage adapté aux enfants ;
- correction claire ;
- barème détaillé.

### Skill : Composition Secondaire Scientifique

- exercices structurés ;
- questions de raisonnement ;
- barème par étape ;
- niveau moyen à difficile ;
- correction détaillée ;
- respect des coefficients.

### Skill : Bilingue Français-Anglais

- consignes en français et en anglais ;
- vocabulaire adapté ;
- questions dans les deux langues ;
- correction bilingue ;
- cohérence terminologique.

### Skill : Maternelle Qualitative

- pas de note chiffrée ;
- activités d’observation ;
- critères qualitatifs ;
- Très Satisfaisant, Satisfaisant, Peu Satisfaisant, Non Satisfaisant ;
- consignes courtes ;
- activités visuelles.

### Skill : Épreuve officielle

- présentation stricte ;
- en-tête établissement ;
- durée ;
- classe ;
- matière ;
- consignes générales ;
- exercices numérotés ;
- barème final ;
- mise en page propre.

## 7.3 Gestion des Skills

L’enseignant peut :

# 8. Workflow complet

## Étape 1 : Choix du type de document

L’enseignant choisit :

- épreuve ;
- devoir ;
- exercice ;
- interrogation ;
- fiche d’activité ;
- sujet d’exposé ;
- sujet de recherche ;
- correction ;
- barème.

## Étape 2 : Ajout des sources

L’enseignant peut :

- téléverser des captures ;
- écrire les grands titres ;
- sélectionner des leçons depuis le cahier journal ;
- sélectionner des notions depuis le cahier de textes ;
- importer un ancien sujet ;
- utiliser une banque de questions.

## Étape 3 : Paramétrage

L’enseignant définit :

- classe ;
- matière ;
- durée ;
- difficulté ;
- nombre d’exercices ;
- note maximale ;
- format ;
- Skill ;
- langue ;
- option bilingue ;
- type de correction.

## Étape 4 : Génération IA

Le système génère :

- le sujet ;
- les consignes ;
- les exercices ;
- le corrigé ;
- le barème ;
- les critères ;
- les observations éventuelles.

## Étape 5 : Modification par l’enseignant

L’enseignant peut :

- modifier le texte ;
- ajouter une question ;
- supprimer une question ;
- reformuler une consigne ;
- changer le barème ;
- ajouter une image ;
- réordonner les exercices ;
- ajuster la difficulté ;
- régénérer une partie seulement.

## Étape 6 : Validation

L’enseignant valide le document.

Selon les règles de l’école, le document peut être :

- sauvegardé directement ;
- soumis à validation du directeur ;
- soumis à validation du responsable pédagogique ;
- marqué comme brouillon ;
- archivé.

## Étape 7 : Sauvegarde

Le document est sauvegardé dans :

- bibliothèque personnelle de l’enseignant ;
- bibliothèque de classe ;
- bibliothèque matière ;
- bibliothèque école si validé ;
- historique pédagogique.

## Étape 8 : Partage et téléchargement

Le document peut être :

- téléchargé en PDF ;
- téléchargé en Word ;
- exporté en Excel si grille/barème ;
- partagé avec un collègue ;
- partagé avec la direction ;
- envoyé aux élèves ;
- envoyé aux parents ;
- imprimé ;
- archivé.

# 16. Sécurité, conformité et garde-fous

Le système doit éviter :

- génération de contenus hors programme ;
- questions inadaptées à l’âge ;
- contenus violents ou sensibles ;
- données personnelles dans les sujets ;
- plagiat non contrôlé ;
- erreurs pédagogiques non vérifiées ;
- sauvegarde sans validation si l’école l’interdit.

Règles :

- l’enseignant valide toujours avant sauvegarde finale ;
- l’IA doit afficher un avertissement : “Document à vérifier par l’enseignant” ;
- les sources doivent être conservées ;
- les versions doivent être historisées ;
- les modifications doivent être tracées ;
- les exports doivent respecter les droits d’accès.

# 20. Conclusion

Le Générateur IA d’Épreuves, Devoirs et Exercices transforme l’Espace Pédagogique Enseignant en véritable studio de production pédagogique.

La chaîne fonctionnelle devient :

1. L’enseignant renseigne les notions étudiées ou importe des captures.
2. Il choisit le type de document.
3. Il applique un Skill pédagogique.
4. Sara AI génère le sujet, le corrigé et le barème.
5. L’enseignant modifie et valide.
6. Le document est sauvegardé.
7. Il peut être partagé, téléchargé, imprimé ou soumis à validation.
8. ORION contrôle la cohérence et signale les anomalies.

Cette fonctionnalité positionne Academia Helm au niveau des solutions modernes augmentées par IA, avec une vraie logique métier scolaire.
