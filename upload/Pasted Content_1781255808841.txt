### Phase 1 : Fondation — Service unifié de double-écriture (1-2 jours)

**Objectif** : Créer un `DualWriteService` qui encapsule le pattern « écrire local + distant simultanément ».

```typescript
// Nouveau fichier : src/lib/offline/dual-write.service.ts

class DualWriteService {
  /**
   * Double-écriture : IndexedDB + API distante simultanément
   * 1. Écrire dans IndexedDB (_isDirty: true) — TOUJOURS
   * 2. Écrire vers l'API distante — immédiatement
   * 3. Si succès distant → _isDirty = false
   * 4. Si échec distant → garder _isDirty, créer event outbox
   */
  async dualWrite(params: {
    storeName: string;
    entityType: SyncEntityType;
    operation: 'CREATE' | 'UPDATE' | 'DELETE';
    data: any;
    tenantId: string;
    apiEndpoint: string;
    method: 'POST' | 'PUT' | 'PATCH' | 'DELETE';
  }): Promise<DualWriteResult>
}
```

**Avantages** :
- Un seul point d'entrée pour TOUS les modules
- Garantie que IndexedDB est TOUJOURS à jour
- L'API distante est appelée immédiatement (pas de fire-and-forget)
- En cas d'échec distant, l'outbox prend le relais

**Livrables** :
- `dual-write.service.ts` avec tests unitaires
- Types `DualWriteResult`, `DualWriteConfig`
- Mécanisme de rollback si l'écriture locale échoue

**Corrections appliquées** (dans `apps/web-app/src/app/jobs/CareersContent.tsx`) :
- ✅ Remplacement de `/api/auth/dev-available-tenants` par `/api/public/schools/with-jobs`

- ✅ Ajout de la priorité school-level > tenant-level pour email et téléphone
